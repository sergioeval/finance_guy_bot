# Política de seguridad

## Reportar una vulnerabilidad

Si descubres una vulnerabilidad de seguridad en este proyecto, por favor **no** la publiques en un issue público.

En su lugar:

1. Abre una **Security Advisory** en la pestaña [Security](https://github.com/sergioeval/finance_guy_bot/security) del repositorio (opción privada), o contacta al mantenedor por otro medio.
2. Incluye pasos para reproducir el problema y el impacto potencial.
3. Permite un tiempo razonable para que se publique un parche antes de divulgar el hallazgo.

## Alcance

Se consideran vulnerabilidades de seguridad, entre otras:

- Exposición de credenciales o datos sensibles
- Inyección de código o SQL
- Acceso no autorizado a datos de otros usuarios
- Fallos que permitan manipular transacciones o saldos

## Respuesta

- Se intentará responder en un plazo de 48 horas.
- Se evaluará el reporte y se trabajará en una solución si procede.
- Se te mantendrá informado sobre el progreso.

## Agradecimientos

Se agradecerá públicamente a quienes reporten vulnerabilidades de forma responsable (si lo desean).
